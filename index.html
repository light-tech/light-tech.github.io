Compiling with DevMax

  1. Go to the app local folder located at %localappdata%\Packages\******.DevMax_****\LocalState (You should open File Explorer, type in %localappdata%\Packages\ and look for some folder with DevMax in the name.) and create 3 folders: projects, include, and lib.
  2. Obtain Windows' C/C++ headers: The easiest way is to install Visual Studio 2017 for Desktop Development. There are 2 set of headers:
    1. Windows SDK headers: Located in C:\Program Files\Windows Kits\10\Include\$VERSION where the current latest $VERSION is 10.0.14393.0. There are 4 folders inside: shared, ucrt, um, and winrt. Usually we only need ucrt which provides those standard C headers i.e. stdio.h and the like but you should copy all of them to the include folder that you created in Step 1.
    2. Microsoft Visual C++ headers: Located in $VISUAL_STUDIO_INSTALL_DIR\VC\Tools\MSVC\$VERSION\include and the $VERSION is something like 14.10.24728. Copy the content of this folder to something like msvc in the include folder that you created in Step 1. This provides you with those C++ headers like iostream.
  3. (OPTIONAL) Copy standard libraries for linking. Again, there are two sets of *.lib files to copy. The first are located in C:\Program Files\Windows Kits\10\Lib\$VERSION\{ucrt,um}\x86 and the second are in $VISUAL_STUDIO_INSTALL_DIR\VC\Tools\MSVC\$VERSION\lib\x86. Just a head-up: You only need kernel32.lib, ucrt.lib in the former and msvcrt.lib and vcruntime.lib in the latter to produce console applications. Just copy them to the lib folder that you created.
  4. In projects: Create folder for each of your project, put your source code there and write build descriptions in project.json. Try cloning our repo as a starting point.
  5. Now you should be able to build the HelloWorld application.

Question & Answers

Q: Why don't you distribute the headers and libraries with the app?

Answer: Copyright issue, of course. We might distribute a non-Microsoft C/C++ standard library in future release.

Q: If we have to install Visual Studio anyway, why should we use DevMax?

Answer: You only need to set it up once, 7-zipped the files (approximately 80MB) and you can use it anywhere. And you don't need to do it in your machine but a virtual machine; which avoid damaging your computer.

Q: Can't you allow headers, libraries and project to be outside of the app's folder?

Answer: Due to severe security restrictions imposed by Microsoft, we must do it that way.
