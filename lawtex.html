LawTeX Guide

LawTeX was designed to be the next generation of TeX and LaTeX IDE:

  • Quick to deploy: No 400MB of download and set-up; packages are fetched as needed.
  • Touch friendly: As tablets and hybrid devices like Microsoft's Surface take over the market, application should integrates well with touch input. Traditional Windows desktop app are not very touch friendly since the menu are hard to click on touchscreen.
  • Efficient: Consume as little memory as possible.
  • High productivity: Automatic recompile and refresh the display when user pauses editing.
  • Reliability: Should not depend on stable Internet connection.
  • Availability on various devices such as PC, tablet, phones, ...

To satisfy these goals, we use Knuth's original TeX program and embrace DVI files instead of PDF files output. This is because DVI are much smaller in size since they do not need to contain the fonts. Knuth's program is known to be pretty much bug-free, compared to more modern engines. It is also much more compact in size.

In this document, we describe the implication of those design choices on how one should use LawTeX.

Conditional Compilation

LawTeX is by no mean designed to replace modern TeX engines such as PDFTeX but to complement them. A typical user uses LawTeX to quickly write or draft the documents and when he/she finishes, he/she then use traditional PDFTeX to produce the print-quality PDF output. Evidently, some LaTeX code might only be applicable to PDFTeX and the final result so one needs a way to turn on/off certain piece of LaTeX code depending on whether we are compiling the final output for publishing or writing drafts. For example, certain PostScript fonts are only available on PDFTeX.

The analogy is developing software in C/C++ where one uses various preprocessor directive #if* ... #endif to conditionally turn on/off codes during development or release.

In LaTeX, this is likewise achieved by definining new \if commands or uses existing \ifPDFTeX .. \fi in the iftex package.

Keyboard

LawTeX has an option to auto-indent code and code suggestion. When you hit Enter, various things can happen. If there is a suggestion (you must enable code suggestion in Settings), LawTeX will insert the first suggestion automatically. If you don't want that behavior, you can use Shift + Enter for a new line. If auto-indent is on and you don't want that, use Ctrl + Enter.

Graphic Support via SVG

The unique feature of LawTeX is its ability to recompile as user pauses. This feature relies on the fact that the compiler can produce DVI output quickly AND the output file can be quicly parsed and rendered. This does not work out well for raster graphic and PDF since they are complicated formats. (Think about having to decode a PNG image over and over again everytime you recompile.)

On the other hand, most illustrations in scientific world come from vector graphic. Thus, we make a choice of supporting vector graphic with SVG by adding new DVI special.
